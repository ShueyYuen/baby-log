# 新生儿照护系统架构设计文档（无 CDN 版本）

---

# 1. 架构目标

## 1.1 设计原则

* 单仓库（Monorepo）
* 前后端分离
* Web 优先
* 简化基础设施
* 保证在中低并发下稳定运行
* 可平滑升级（未来可引入 CDN）

---

## 1.2 架构约束

* 使用数据库 + 对象存储（S3/OSS）
* 系统运行在浏览器
* 支持图片上传与访问
* 支持家庭协作

---

---

# 2. 总体架构

---

## 2.1 架构结构

```text
[ Browser ]
     ↓
[ Web Server (静态资源 + API入口) ]
     ↓
[ API Application ]
     ↓
[ Database ]

[ Object Storage (S3/OSS) ]
```

---

## 2.2 关键变化（相对 CDN 架构）

| 项目   | CDN架构 | 当前架构          |
| ---- | ----- | ------------- |
| 静态资源 | CDN分发 | Web Server 提供 |
| 图片访问 | CDN加速 | 直接访问 S3 / 代理  |
| 缓存   | CDN缓存 | 浏览器 + 服务器缓存   |

---

## 4.1 部署方式

* HTML / JS / CSS

---

## 4.2 静态资源加载策略（关键）

由于没有 CDN，需要优化：

### 4.2.1 缓存策略

* 文件名带 hash（强缓存）
* HTTP Header：

  * Cache-Control: max-age=31536000（静态资源）
  * HTML：no-cache

---

### 4.2.2 资源优化

* 代码分包（chunk splitting）
* 按需加载（lazy loading）
* 压缩（gzip / brotli）

# 6. 图片存储与访问（重点修改）

---

## 6.1 存储位置

* 使用 S3 / OSS 存储图片

---

## 6.2 访问模式（两种可选）

---

### 方案 A：直接访问对象存储（推荐）

```text
Browser → S3 URL
```

#### 特点：

* 简单
* 性能好（绕过后端）
* 依赖对象存储公网能力

---

### 方案 B：后端代理访问

```text
Browser → API → S3
```

#### 特点：

* 更安全（可控权限）
* 后端增加带宽压力

---

## 6.3 推荐策略

* 默认使用 **直接访问 S3**
* 敏感数据使用 **签名 URL**

# 8. Web Server 设计（替代 CDN 的关键）

---

## 8.1 职责

* 提供前端静态资源
* 作为 API 网关入口（反向代理）
* 提供基础缓存能力

---

## 8.2 推荐结构

```text
Nginx
 ├── /        → 静态资源（前端）
 └── /api     → 转发到 API Server
```

---

## 8.3 缓存策略

### 静态资源

* 强缓存（1年）
* 文件名 hash

---

### API

* 不缓存（或短缓存）

---

# 9. 性能设计（无 CDN 重点）

---

## 9.1 前端优化

* 懒加载页面
* 图片懒加载
* 限制首屏 JS 体积

---

## 9.2 图片优化（关键）

必须做：

* 前端压缩图片
* 限制图片尺寸
* 控制上传质量

否则：

👉 带宽会成为瓶颈

---

## 9.3 后端优化

* 分页查询（时间线）
* 限制返回字段
* 避免大 JSON

---

---

# 10. 网络与带宽策略

---

## 10.1 问题

无 CDN 时：

* 静态资源 → 直接来自服务器
* 图片 → 直接来自 S3

---

## 10.2 优化手段

* 使用就近区域的 S3
* 减少图片体积
* 减少请求数量（合并资源）

---

---

## 11.1 图片访问控制

## 11.2 API 安全

# 12. 部署架构

---

## 12.1 部署结构

```text
[ Browser ]
     ↓
[ Nginx / Web Server ]
     ↓            ↓
[ Static Files ]  [ API Server ]
                      ↓
                 [ Database ]

[ S3 / OSS ]
```

---

## 12.2 部署建议

* 前端 + Nginx 同机部署（小规模）
* API 单独容器
* 数据库托管服务
